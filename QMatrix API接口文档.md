# QMatrix API 接口文档

## 1. 接口概览

QMatrix API 提供了与QMatrix后端服务交互的能力，支持对话管理、历史记录查询、流式响应等功能。本文档详细描述了所有可用的API接口，包括请求参数、响应格式、错误处理等信息。

## 2. 接口列表

| 接口名称 | 路径 | 请求方法 | 功能描述 |
|---------|------|---------|----------|
| 获取历史记录 | /api/history | GET | 获取所有对话历史记录 |
| 获取对话详情 | /api/history/{conversationId} | GET | 获取指定对话的详细消息 |
| 发送消息 | /api/dialogue | POST | 发送消息到指定对话 |
| 流式响应 | /api/stream | POST | 获取流式响应 |
| 获取Agent进度 | /api/agent/progress/{conversationId} | GET | 获取Agent执行进度 |

## 3. 接口详细说明

### 3.1 获取历史记录

**接口基本信息**
- **接口名称**：获取历史记录
- **路径**：/api/history
- **请求方法**：GET
- **版本号**：v1

**功能描述**
获取用户的所有对话历史记录，包括对话ID、标题和更新时间等信息。

**请求参数**
| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|---------|----------|---------|------|
| 无 | - | - | - | 无请求参数 |

**响应数据结构**
| 字段名 | 数据类型 | 说明 |
|--------|---------|------|
| id | string | 对话ID |
| title | string | 对话标题 |
| createdAt | string | 创建时间（ISO格式） |
| updatedAt | string | 更新时间（ISO格式） |

**错误码说明**
| 错误码 | 描述 |
|--------|------|
| 500 | 服务器内部错误 |

**请求示例**
```http
GET /api/history
Content-Type: application/json
```

**响应示例**
```json
[
  {
    "id": "123456",
    "title": "关于项目计划的讨论",
    "createdAt": "2026-03-21T10:00:00Z",
    "updatedAt": "2026-03-21T10:30:00Z"
  },
  {
    "id": "789012",
    "title": "技术问题咨询",
    "createdAt": "2026-03-20T14:00:00Z",
    "updatedAt": "2026-03-20T14:15:00Z"
  }
]
```

**接口权限要求**
- 无特殊权限要求

**调用频率限制**
- 无限制

**特殊说明**
- 返回的历史记录按更新时间倒序排列

### 3.2 获取对话详情

**接口基本信息**
- **接口名称**：获取对话详情
- **路径**：/api/history/{conversationId}
- **请求方法**：GET
- **版本号**：v1

**功能描述**
获取指定对话的详细消息列表，包括用户消息和Agent回复。

**请求参数**
| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|---------|----------|---------|------|
| conversationId | string | 是 | - | 对话ID |

**响应数据结构**
| 字段名 | 数据类型 | 说明 |
|--------|---------|------|
| role | string | 角色（user/agent） |
| content | string | 消息内容 |
| timestamp | string | 消息时间（ISO格式） |
| type | string | 消息类型（text/agent_step） |
| isStreaming | boolean | 是否正在流式传输 |
| agentSteps | array | Agent步骤列表（仅当type为agent_step时） |

**错误码说明**
| 错误码 | 描述 |
|--------|------|
| 404 | 对话不存在 |
| 500 | 服务器内部错误 |

**请求示例**
```http
GET /api/history/123456
Content-Type: application/json
```

**响应示例**
```json
[
  {
    "role": "user",
    "content": "你好，我有一个技术问题",
    "timestamp": "2026-03-21T10:00:00Z",
    "type": "text",
    "isStreaming": false
  },
  {
    "role": "agent",
    "content": "您好，请问有什么可以帮助您的？",
    "timestamp": "2026-03-21T10:00:05Z",
    "type": "text",
    "isStreaming": false
  }
]
```

**接口权限要求**
- 无特殊权限要求

**调用频率限制**
- 无限制

**特殊说明**
- 返回的消息按时间顺序排列

### 3.3 发送消息

**接口基本信息**
- **接口名称**：发送消息
- **路径**：/api/dialogue
- **请求方法**：POST
- **版本号**：v1

**功能描述**
向指定对话发送消息，触发Agent处理。

**请求参数**
| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|---------|----------|---------|------|
| conversation_id | string | 是 | - | 对话ID |
| message | string | 是 | - | 消息内容 |

**响应数据结构**
| 字段名 | 数据类型 | 说明 |
|--------|---------|------|
| status | string | 处理状态 |
| message | string | 处理结果消息 |

**错误码说明**
| 错误码 | 描述 |
|--------|------|
| 400 | 请求参数错误 |
| 404 | 对话不存在 |
| 500 | 服务器内部错误 |

**请求示例**
```http
POST /api/dialogue
Content-Type: application/json

{
  "conversation_id": "123456",
  "message": "如何实现一个聊天机器人？"
}
```

**响应示例**
```json
{
  "status": "success",
  "message": "消息已接收，正在处理"
}
```

**接口权限要求**
- 无特殊权限要求

**调用频率限制**
- 每对话每分钟最多10次请求

**特殊说明**
- 此接口仅接收消息，不返回Agent的回复
- Agent的回复需要通过流式响应接口获取

### 3.4 流式响应

**接口基本信息**
- **接口名称**：流式响应
- **路径**：/api/stream
- **请求方法**：POST
- **版本号**：v1

**功能描述**
获取Agent的流式响应，适用于实时显示Agent的回复过程。

**请求参数**
| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|---------|----------|---------|------|
| conversation_id | string | 是 | - | 对话ID |
| message | string | 是 | - | 消息内容 |
| stream | boolean | 是 | true | 是否启用流式响应 |

**响应数据结构**
- 流式响应，每行数据格式为：`data: <内容>`
- 结束标记为：`data: [DONE]`

**错误码说明**
| 错误码 | 描述 |
|--------|------|
| 400 | 请求参数错误 |
| 404 | 对话不存在 |
| 500 | 服务器内部错误 |

**请求示例**
```http
POST /api/stream
Content-Type: application/json

{
  "conversation_id": "123456",
  "message": "如何实现一个聊天机器人？",
  "stream": true
}
```

**响应示例**
```
data: 实现一个聊天机器人需要以下步骤：
data: 1. 确定机器人的功能和用途
data: 2. 选择合适的技术栈
data: 3. 设计对话流程
data: 4. 实现自然语言处理
data: 5. 部署和测试
data: [DONE]
```

**接口权限要求**
- 无特殊权限要求

**调用频率限制**
- 每对话每分钟最多5次请求

**特殊说明**
- 响应为服务器发送事件（Server-Sent Events）格式
- 客户端需要处理流式数据，逐段显示内容

### 3.5 获取Agent进度

**接口基本信息**
- **接口名称**：获取Agent进度
- **路径**：/api/agent/progress/{conversationId}
- **请求方法**：GET
- **版本号**：v1

**功能描述**
获取Agent在指定对话中的执行进度，包括当前步骤、状态和详细信息。

**请求参数**
| 参数名 | 数据类型 | 是否必填 | 默认值 | 说明 |
|--------|---------|----------|---------|------|
| conversationId | string | 是 | - | 对话ID |

**响应数据结构**
| 字段名 | 数据类型 | 说明 |
|--------|---------|------|
| stepId | string | 步骤ID |
| name | string | 步骤名称 |
| status | string | 步骤状态（pending/running/completed/failed） |
| progress | number | 进度百分比（0-100） |
| details | object | 步骤详细信息 |
| startTime | string | 开始时间（ISO格式） |
| endTime | string | 结束时间（ISO格式，仅当完成时） |

**错误码说明**
| 错误码 | 描述 |
|--------|------|
| 404 | 对话不存在 |
| 500 | 服务器内部错误 |

**请求示例**
```http
GET /api/agent/progress/123456
Content-Type: application/json
```

**响应示例**
```json
{
  "stepId": "step_1",
  "name": "分析用户请求",
  "status": "running",
  "progress": 50,
  "details": {
    "currentTask": "提取关键词",
    "completedTasks": ["解析输入"]
  },
  "startTime": "2026-03-21T10:00:00Z"
}
```

**接口权限要求**
- 无特殊权限要求

**调用频率限制**
- 每对话每秒最多1次请求

**特殊说明**
- 仅在Agent执行任务时返回有意义的进度信息
- 当Agent未执行任务时，返回默认空进度

## 4. 通用错误处理

| 错误码 | 状态码 | 描述 | 处理建议 |
|--------|--------|------|----------|
| 400 | Bad Request | 请求参数错误 | 检查请求参数是否符合要求 |
| 401 | Unauthorized | 未授权 | 检查认证信息 |
| 403 | Forbidden | 禁止访问 | 检查权限设置 |
| 404 | Not Found | 资源不存在 | 检查请求路径和资源ID |
| 429 | Too Many Requests | 请求过于频繁 | 减少请求频率 |
| 500 | Internal Server Error | 服务器内部错误 | 联系服务器管理员 |

## 5. 接口调用最佳实践

1. **错误处理**：总是处理API返回的错误，尤其是网络错误和服务器错误
2. **重试机制**：对于临时性错误（如网络超时），实现适当的重试机制
3. **流式响应**：使用流式响应接口获取Agent回复，提供更好的用户体验
4. **缓存策略**：对历史记录等静态数据实现客户端缓存，减少API调用
5. **批量操作**：避免频繁调用API，尽量批量获取数据
6. **参数验证**：在客户端对请求参数进行验证，减少无效请求
7. **超时设置**：为API调用设置合理的超时时间，避免长时间阻塞
